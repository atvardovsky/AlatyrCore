## Required Target Actions

- Do not copy this release over an installed adapter without a reviewed target
  migration assessment.
- Preserve target-owned project facts, commands, deviations, approvals, and
  source-of-truth decisions.
- Compare the installed framework baseline and template surfaces with this
  report, then identify the target-specific repair set.
- Obtain explicit approval before overwriting existing AI instructions or
  changing protected adapter behavior.
- Update the target manifest framework and template versions only after the
  corresponding files and local deviations have been reviewed.
- Run target validation or record unresolved checks and residual risks.

## Optional Target Actions

- Move an installation to the `core`, `standard`, or `full` scaffold profile
  that matches its maintained modules.
- Enable large-task orchestration, consistency mapping, or AI infrastructure
  routing only when the project needs and can maintain those modules.
- Capture target effectiveness metrics before making project-specific cost or
  quality claims.

## Residual Risks

- Native macOS and Windows results depend on the published GitHub Actions run.
- Non-Codex assistant surfaces have static bridge and prepared-run conformance,
  but no fabricated vendor execution evidence is claimed.
- The controlled docs-local benchmark does not establish a general token-cost
  reduction for every task or model.
- Target adapters may contain local deviations not visible from this source
  comparison.
